# 编辑模式按钮修复完成

## ✅ 问题已修复

### 🐛 问题原因

1. **预览模式的"编辑模式"按钮没有事件处理**
   - 按钮点击后发送了消息，但扩展没有处理

2. **package.json 中没有注册编辑器模式**
   - 只注册了预览模式，编辑器模式无法通过 "Open With" 菜单打开

---

### 🔧 修复内容

#### 1. **预览模式添加消息处理**

在 `wpsPreviewProvider.ts` 中添加：

```typescript
// 监听来自 webview 的消息
webviewPanel.webview.onDidReceiveMessage(async (message) => {
  switch (message.type) {
    case "edit":
      // 切换到编辑器模式
      await vscode.commands.executeCommand(
        "vscode.openWith",
        document.uri,
        "wpsEditor.editor",
      );
      break;
  }
});
```

**功能**：
- ✅ 点击"编辑模式"按钮后，自动切换到编辑器模式
- ✅ 无需手动关闭预览窗口

#### 2. **package.json 添加编辑器模式注册**

```json
{
  "customEditors": [
    {
      "viewType": "wpsEditor.preview",
      "displayName": "WPS Editor",
      "priority": "default"
    },
    {
      "viewType": "wpsEditor.editor",
      "displayName": "WPS Editor (Editor)",
      "priority": "option"
    }
  ]
}
```

**功能**：
- ✅ 编辑器模式出现在 "Open With" 菜单中
- ✅ 可以通过右键菜单打开编辑器模式

## 🎯 测试步骤

### 方法 1：通过"编辑模式"按钮切换（推荐）

1. **重新加载窗口**
   - 按 **Ctrl+Shift+P**
   - 输入 **"Developer: Reload Window"**
   - 回车执行

2. **打开预览模式**
   - 右键 .docx 文件
   - 选择 **"Open With..."**
   - 选择 **"WPS Editor"**

3. **点击"编辑模式"按钮**
   - 点击工具栏上的 **"编辑模式"** 按钮
   - 自动切换到编辑器模式

4. **测试编辑**
   - 点击编辑器区域
   - 输入文字
   - 点击 **B** 按钮加粗

---

### 方法 2：直接打开编辑器模式

1. **重新加载窗口**
   - 按 **Ctrl+Shift+P**
   - 输入 **"Developer: Reload Window"**
   - 回车执行

2. **打开编辑器模式**
   - 右键 .docx 文件
   - 选择 **"Open With..."**
   - 选择 **"WPS Editor (Editor)"** ⬅️ 注意有 "(Editor)"

3. **测试编辑**
   - 点击编辑器区域
   - 输入文字
   - 点击 **B** 按钮加粗

#### 预览 → 编辑器
- 点击"编辑模式"按钮 ✅

- 点击"👁️ 预览"按钮
- 或关闭编辑器，重新用预览模式打开

在编辑器模式中：
- **Ctrl+S** - 保存
- **Ctrl+B** - 加粗
- **Ctrl+I** - 斜体
- **Ctrl+U** - 下划线
