# 快速测试指南

## 🚀 立即测试

### 1. 编译项目
```bash
npm run compile
```
✅ 编译成功

### 2. 启动调试
按 **F5** 或运行：
```bash

```

### 3. 测试 .docx 文件（无需 LibreOffice）

#### 步骤：
1. 准备一个测试 .docx 文件
2. 在 VSCode 中打开
3. 右键 → "Open With" → "WPS Editor (Editor)"

#### 预期结果：
✅ **编辑器打开**
- 工具栏显示完整格式化工具
- 文档内容可编辑
- 底部显示 "JavaScript 引擎（无需 LibreOffice）"

✅ **编辑功能**
- 选中文本，点击 **B** → 文本加粗
- 选中文本，点击 **I** → 文本斜体
- 选中文本，点击 **U** → 添加下划线
- 选择标题样式 → 应用标题格式

✅ **分页功能**
- 底部显示 "第 1 页 / 共 X 页"
- 点击 ◀️ ▶️ 可以翻页
- 滚动时页码自动更新

✅ **字数统计**
- 右下角显示 "字数：XX"
- 编辑时实时更新

✅ **保存功能**
- 点击 "💾 保存" 或按 **Ctrl+S**
- 显示 "文档已保存" 提示
- 文件内容已更新

## 🎯 快速验证脚本

创建一个测试文档：

```javascript
// test.js - 快速创建测试文档
const fs = require('fs');

// 创建简单的 HTML 测试内容
const testContent = `
<html>
<body>
  <h1>测试文档</h1>
  <p>这是一个<strong>测试</strong>文档。</p>
  <ul>
    <li>项目 1</li>
    <li>项目 2</li>
    <li>项目 3</li>
  </ul>
  <h2>表格测试</h2>
  <table border="1">
    <tr><th>姓名</th><th>年龄</th></tr>
    <tr><td>张三</td><td>25</td></tr>
    <tr><td>李四</td><td>30</td></tr>
  </table>
</body>
</html>
`;

fs.writeFileSync('test.html', testContent);
console.log('测试文件已创建：test.html');
```

## 🐛 常见问题

### Q1: 打开文件显示 "加载文档失败"
**A**: 检查文件格式是否支持
- ✅ 支持：.docx, .xlsx, .wps, .et, .dps
- ❌ 不支持：其他格式

### Q2: 格式化按钮无效
**A**: 确保选中了文本
- 选中要格式化的文本
- 然后点击格式化按钮

### Q3: 保存后文件没有变化
**A**: 检查文件权限
- 确保文件没有被其他程序占用
- 确保有写入权限

### Q4: 分页不准确
**A**: 分页基于估算（800px/页）
- 可以手动插入分页符调整
- 这是已知限制，后续会优化
